# Cursor AI 标准提示语规范文档

## 1. 概述

本文档定义了在使用 Cursor AI 进行开发时，编写有效提示语的标准规范。遵循这些规范可以显著提高 AI 助手的响应质量和代码准确性。

## 2. 基本原则

### 2.1 明确性原则
- **明确目标**：清晰说明需要完成的任务
- **具体描述**：避免模糊和抽象的表达
- **提供上下文**：包含必要的背景信息

### 2.2 结构化原则
- **分步骤描述**：将复杂任务分解为多个步骤
- **使用列表**：使用有序列表或无序列表组织信息
- **层次清晰**：使用标题和段落组织内容

### 2.3 完整性原则
- **包含所有需求**：一次性提供完整的需求信息
- **明确边界**：说明什么需要做，什么不需要做
- **指定约束**：明确技术约束和限制条件

## 3. 提示语结构

### 3.1 标准结构

```
[任务类型] [目标描述]

**上下文**：
- 项目/功能背景
- 相关代码位置
- 依赖关系

**需求**：
- 具体功能点1
- 具体功能点2
- ...

**约束条件**：
- 技术约束
- 业务规则
- 性能要求

**验收标准**：
- 功能验收标准
- 代码质量标准
```

### 3.2 任务类型标签

- `[功能实现]` - 实现新功能
- `[问题修复]` - 修复 Bug
- `[代码重构]` - 重构现有代码
- `[性能优化]` - 优化性能
- `[文档更新]` - 更新文档
- `[测试编写]` - 编写测试代码
- `[代码审查]` - 审查代码质量

## 4. 提示语编写规范

### 4.1 功能实现类提示语

**模板**：
```
[功能实现] 实现 [功能名称]

**功能描述**：
详细描述要实现的功能，包括：
- 功能用途
- 用户交互流程
- 预期行为

**技术实现**：
- 使用的技术栈
- 架构模式
- 数据模型

**相关文件**：
- @文件路径1
- @文件路径2

**参考**：
- 类似功能的实现
- 官方文档链接
```

**示例**：
```
[功能实现] 实现用户登录功能

**功能描述**：
用户通过用户名和密码登录系统，登录成功后跳转到主界面。

**技术实现**：
- 使用 MVVM 模式
- 密码使用 SHA256 加密
- 登录状态存储在 Session 中

**相关文件**：
- @src/Hbt.Fluent/Views/Login/LoginView.xaml
- @src/Hbt.Fluent/ViewModels/Login/LoginViewModel.cs
- @src/Hbt.Application/Services/Identity/IAuthenticationService.cs

**参考**：
- 项目中的 AuthenticationService 实现
```

### 4.2 问题修复类提示语

**模板**：
```
[问题修复] 修复 [问题描述]

**问题描述**：
- 问题现象
- 重现步骤
- 错误信息

**相关代码**：
- @文件路径（问题所在位置）

**日志/错误信息**：
```
错误堆栈信息
```

**预期行为**：
描述修复后应该表现的行为

**约束条件**：
- 不能影响现有功能
- 必须遵循项目规范
```

**示例**：
```
[问题修复] 修复标签页图标不显示的问题

**问题描述**：
- 问题现象：标签页标题显示正常，但图标不显示
- 重现步骤：点击菜单项，打开新的标签页
- 错误信息：无错误，但图标不显示

**相关代码**：
- @src/Hbt.Fluent/Views/MainWindow.xaml
- @src/Hbt.Fluent/Models/AvalonDocument.cs
- @src/Hbt.Fluent/ViewModels/MainWindowViewModel.cs

**日志信息**：
```
[AvalonDocument] 图标设置成功: Icon=House, IconSource=RenderTargetBitmap, Width=16, Height=16, IconSourceIsNull=False
```

**预期行为**：
标签页应显示：图标 + 标题 + 关闭按钮

**约束条件**：
- 必须基于真实的代码给出结论
- 不能破坏现有的标题显示功能
```

### 4.3 代码重构类提示语

**模板**：
```
[代码重构] 重构 [模块/功能名称]

**重构目标**：
- 提高代码可读性
- 消除重复代码
- 优化架构设计

**重构范围**：
- @文件路径1
- @文件路径2

**重构原则**：
- 遵循 SOLID 原则
- 保持功能不变
- 遵循项目命名规范

**验收标准**：
- 代码通过所有测试
- 代码审查通过
- 性能不降低
```

### 4.4 性能优化类提示语

**模板**：
```
[性能优化] 优化 [功能/模块] 的性能

**性能问题**：
- 当前性能指标
- 性能瓶颈分析
- 性能测试数据

**优化目标**：
- 响应时间目标
- 资源使用目标
- 吞吐量目标

**优化策略**：
- 缓存策略
- 异步处理
- 数据库优化

**验收标准**：
- 性能指标达到目标
- 功能正常
- 代码质量不降低
```

## 5. 特殊场景提示语

### 5.1 基于现有代码的修改

```
[功能实现] 基于现有代码实现 [功能名称]

**现有代码位置**：
- @文件路径

**需要修改的内容**：
1. 修改点1：具体描述
2. 修改点2：具体描述

**保持不变的代码**：
- 不需要修改的部分

**参考标准**：
- 项目中的类似实现
- 官方文档
```

### 5.2 遵循项目规范的提示语

```
[功能实现] 实现 [功能名称]（遵循项目规范）

**项目规范要求**：
- 命名规范：类名以 Hbt 开头，使用 PascalCase
- 代码组织：遵循分层架构
- 注释规范：必须包含 XML 注释

**相关规范文件**：
- @项目规范文档路径

**必须遵循**：
- 后端命名规范
- 前端命名规范
- 代码组织规范
```

### 5.3 需要验证的提示语

```
[功能实现] 实现 [功能名称]（需要验证）

**实现步骤**：
1. 实现功能
2. 添加日志记录
3. 提供验证方法

**验证要求**：
- 必须提供日志输出
- 必须提供测试步骤
- 必须基于真实代码验证

**不接受**：
- 推测性结论
- 未经验证的假设
- 模棱两可的答案
```

## 7. 提示语中的约束条件表达

### 7.1 必须遵循

- **必须**：强制要求
- **禁止**：不允许的行为
- **只能**：唯一的选项

### 7.2 推荐遵循

- **应该**：推荐的做法
- **建议**：可选的改进
- **最好**：最佳实践

### 7.3 禁止事项

- **不要**：明确禁止的行为
- **避免**：尽量避免的做法
- **禁止**：严格禁止

## 8. 提示语示例集合

### 8.1 完整示例：功能实现

```
[功能实现] 实现用户管理页面的数据分页功能

**功能描述**：
在用户管理页面中实现数据分页显示，支持：
- 每页显示数量选择（10/20/50/100）
- 上一页/下一页导航
- 跳转到指定页码
- 显示总记录数和当前页信息

**技术实现**：
- 使用 MVVM 模式
- 使用 DataGrid 控件显示数据
- 后端 API 支持分页参数（pageIndex, pageSize）
- 前端 ViewModel 管理分页状态

**相关文件**：
- @src/Hbt.Fluent/Views/Identity/UserView.xaml
- @src/Hbt.Fluent/ViewModels/Identity/UserViewModel.cs
- @src/Hbt.Application/Services/Identity/IUserService.cs

**UI 要求**：
- 分页控件位于 DataGrid 底部
- 使用 Fluent Design 风格
- 支持键盘导航

**约束条件**：
- 必须遵循项目命名规范（Hbt 前缀，PascalCase）
- 必须添加 XML 注释
- 必须处理异常情况（空数据、网络错误等）

**验收标准**：
- 分页功能正常工作
- UI 符合设计规范
- 代码通过代码审查
```

### 8.2 完整示例：问题修复

```
[问题修复] 修复标签页图标不显示的问题

**问题描述**：
- 问题现象：标签页标题显示正常，但图标不显示
- 重现步骤：
  1. 点击左侧菜单树中的菜单项
  2. 打开新的标签页
  3. 观察标签页头部，只有标题和关闭按钮，没有图标
- 错误信息：无编译错误，无运行时错误

**相关代码**：
- @src/Hbt.Fluent/Views/MainWindow.xaml（DockingManager 配置）
- @src/Hbt.Fluent/Models/AvalonDocument.cs（图标转换和设置）
- @src/Hbt.Fluent/ViewModels/MainWindowViewModel.cs（文档创建逻辑）

**日志信息**：
```
[AvalonDocument] 图标设置成功: Icon=House, IconSource=RenderTargetBitmap, Width=16, Height=16, IconSourceIsNull=False
[AvalonDock] IconSource 验证: Icon=House, IconSource=RenderTargetBitmap, Width=16, Height=16
```

**分析要求**：
- 必须基于真实的代码给出结论，不能给出模棱两可的答案
- 必须亲自核对代码，保证结论百分之百正确
- 必须明确信息的来源（来自代码还是推测）

**预期行为**：
标签页应显示：图标 + 标题 + 关闭按钮

**约束条件**：
- 不能破坏现有的标题显示功能
- 不能影响关闭按钮的显示
- 必须保证原有功能代码正常运行

**参考**：
- AvalonDock 官方文档：https://github.com/Dirkster99/AvalonDock/wiki
- 项目中的类似实现
```

## 9. 提示语质量检查清单

- [ ] 任务类型标签是否明确

- [ ] 是否提供了相关代码路径
- [ ] 是否明确约束条件
- [ ] 是否定义了验收标准
- [ ] 是否遵循项目规范
- [ ] 是否避免了模糊表达
- [ ] 是否提供了足够的上下文

## 10. 常见错误和避免方法

### 10.1 错误示例

❌ **不好的提示语**：
```
修复图标问题
```

✅ **改进后的提示语**：
```
[问题修复] 修复标签页图标不显示的问题

**问题描述**：
标签页标题显示正常，但图标不显示。

**相关代码**：
- @src/Hbt.Fluent/Views/MainWindow.xaml
- @src/Hbt.Fluent/Models/AvalonDocument.cs

**预期行为**：
标签页应显示：图标 + 标题 + 关闭按钮
```

### 10.2 常见问题

1. **缺少上下文**：没有提供足够的背景信息
   - 解决方法：明确说明项目背景、相关代码位置

2. **需求不明确**：功能需求描述模糊
   - 解决方法：使用具体的用户场景描述需求

3. **缺少约束**：没有说明技术约束和业务规则
   - 解决方法：明确列出所有约束条件

4. **没有验收标准**：无法判断任务是否完成
   - 解决方法：定义明确的验收标准

## 11. 最佳实践

### 11.1 提示语编写最佳实践

1. **使用结构化格式**：遵循标准模板
2. **提供代码引用**：使用 @ 符号引用相关文件
3. **明确验收标准**：定义可验证的完成标准
4. **包含上下文信息**：提供足够的背景信息
5. **使用具体示例**：提供代码示例或参考实现

### 11.2 与 AI 交互最佳实践

1. **迭代改进**：根据 AI 的响应调整提示语
2. **提供反馈**：明确说明哪些部分正确，哪些需要修改
3. **验证结果**：要求 AI 提供验证方法或测试步骤
4. **保持一致性**：使用统一的术语和格式
